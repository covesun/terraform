# Security Rules For Agent-Driven IaC

AI エージェント駆動の構築運用で守る共通ルール。

## 1. 平文シークレットをファイルに保存しない
- `terraform.tfvars` や Markdown にパスワードを記載しない。
- シークレットは実行時注入 (`TF_VAR_*`) を使う。

## 2. 長期鍵より短期トークンを優先する
- 固定アクセスキーより Managed Identity / OIDC を優先する。
- 例外で固定鍵を使う場合は期限と用途を明記する。

## 3. 最小権限を徹底する
- エージェント実行権限は必要最小限にする。
- `Owner` 常用を避け、目的単位でロールを分離する。

## 4. 資格情報は使い捨て前提で管理する
- 検証用シークレットは短命化し、定期ローテーションする。
- 実験終了後は失効または環境破棄する。

## 5. ログに秘密情報を残さない
- コマンド出力、Execution Log、チャットにシークレットを残さない。
- 必要な共有時は必ずマスクする。

## 6. 破壊的操作には承認ゲートを入れる
- `apply`/`destroy` や高権限操作は人間承認後に実行する。
- 承認結果を `tasks.md` の記録に残す。

## 7. 本番環境と検証環境を分離する
- AI検証は専用サブスクリプション/アカウントで実施する。
- 本番への直実行を禁止する。

## 8. 監査可能性を確保する
- 誰が・いつ・何を実行したか追跡可能にする。
- `tasks.md` の Execution Log とクラウド監査ログを併用する。
